④ 光ファイバ増幅器内部で生ずる自然放出光と信号光とによって生ずるビート雑音は、出力側においてSN比を低下させる要因となる。

**【解答】**
（オ） **①**

**【解説】**
> この問題は、光通信システムにおける雑音の種類と性質について問うています。

IMSは、セッション制御用のプロトコルとしてIP電話などで用いられるSIPを用いて通信路の設定・解放を制御し、マルチメディアサービスを提供している。また、ユーザ認証などのアクセスセキュリティ管理、ローミング対応のための **（ア）** 管理、アプリケーションサーバ(AS)との連携、課金管理機能へのインタフェース提供などの機能を有している。
IMSのアーキテクチャの重要な構成要素であるCSCFは、SIPサーバに相当する機能を持ち、CSCFにはP-CSCF、S-CSCF及びI-CSCFがある。

### (2)
トラヒックの用語などについて述べた次の文章のうち、正しいものは、（エ）である。

**＜（エ）の解答群＞**
① 1日のうちトラヒックの最大となる連続する1時間は最繁時といわれ、1日の全トラヒックに対する最繁時トラヒックの占める割合は、基礎トラヒックといわれる。
② 即時式の系において、加えられた呼量と運ばれた呼量との差である損失呼量の運ばれた呼量に対する比は、呼損率といわれる。
③ ある測定時間T〔時間〕中における総呼数をN〔呼〕、平均保留時間をt〔時間〕とすると、呼量A〔アーラン〕は次式で表される。
A = Nt / T
④ 待時式完全群において、加わる呼量を一定とした場合、出回線数が多くなれば平均待合せ時間は長くなる。

**【解答】**
（エ） **③**

**【解説】**
> この問題は、トラヒック理論に関する基本的な用語と公式の正誤を問うています。

*   ① （誤り）： 最繁時（Busy Hour）の定義は正しいですが、「基礎トラヒック」の定義が誤りです。基礎トラヒックとは、最繁時以外の時間帯における平均的なトラヒック量を指すことが一般的です。
*   ② （誤り）： 呼損率（Blocking Probability / Call Loss Rate）は、損失呼量を**加えられた呼量**で割った比率です。（損失呼量）/（加えられた呼量）。記述の「運ばれた呼量に対する比」ではありません。
*   **③ （正しい）**: 呼量（Traffic Volume）の定義式として正しいです。呼量 A (アーラン)は、測定時間 T の間に発生した全ての呼の保留時間の総和（N × t）を測定時間 T で割ることで求められます。これは、回線を平均していくつの呼が同時に使用していたかを示す値です。
*   ④ （誤り）： 待時式完全群（待ち行列モデル）において、加わる呼量が一定の場合、サービスを提供する出回線数が多くなれば、呼が処理される機会が増えるため、平均待合せ時間は**短く**なります。

*   **B （正しい）**: 予防保全の代表的な手法である時間計画保全（TBM: Time Based Maintenance）の分類を正しく説明しています。一定時間ごと（例：1年ごと）に行うのが定期保全、一定の稼働時間ごと（例：1000時間運転ごと）に行うのが経時保全です。
*   **C （正しい）**: 予防保全のもう一つの主要な手法が状態監視保全（CBM: Condition Based Maintenance）です。設備の振動、温度、油の劣化などをセンサーで常時監視し、異常の兆候（故障の予兆）を検知した時点で保全を行います。これにより、不要な保全を減らし、部品の寿命を最大限に活用できます。摩耗故障など、故障発生が予測しにくい場合に特に有効です。

したがって、A、B、Cはいずれも正しい記述です。

### (5)
内閣府から公表されている事業継続ガイドライン(令和5年3月)における事業継続マネジメント(BCM)の概要などについて述べた次の文章のうち、誤っているものは、（ク）である。

**＜（ク）の解答群＞**
① BCMは、事業継続計画(BCP)策定や維持・更新、事業継続を実現するための予算・資源の確保、事前対策の実施、取組を浸透させるための教育・訓練の実施、点検、継続的な改善などを行う平常時からのマネジメント活動とされている。
② 企業・組織は、様々な危機的な発生事象に直面しても、利害関係者から、重要な事業の継続又は早期の復旧を望まれているため、このような利害関係者のニーズと期待を十分に認識し、BCMを積極的に経営戦略に反映すべきである。
③ BCMは、企業・組織全体のマネジメントとして継続的・体系的に取り組むことが重要である。その手法として、例えば、PDCAサイクルなどのマネジメントに関する仕組みの活用も有効である。
④ BCMは、どのような危機的な事象が発生しても重要業務を継続するという目的で実施するものであることを考慮することが重要である。この点から、BCMでは、自社に生じた事態を結果事象(例えば、自社の〇〇拠点が使用不能)により考えるのではなく、原因事象(例えば、直下型地震)により考え、対応策を検討することが推奨される。

**【解答】】**
（ク） **④**

**【解説】】**
> この問題は、BCM（事業継続マネジメント）の基本的な考え方に関する正誤問題です。

*   ① （正しい）： BCMは、BCPという計画書を作って終わりではなく、それを実効性のあるものにするためのPDCAサイクル（計画→実行→評価→改善）を継続的に回していく、平常時からのマネジメント活動全体を指します。
*   ② （正しい）： BCMは、単なる防災対策ではなく、企業の社会的責任や事業価値を維持・向上させるための重要な経営戦略の一つとして位置づけられています。顧客、取引先、株主、従業員といったステークホルダー（利害関係者）からの信頼を確保するために不可欠です。
*   ③ （正しい）： BCMは、一部の部署だけでなく、経営層のリーダーシップのもとで全社的に取り組むべき活動です。その継続的な改善のためには、品質マネジメントなどでも用いられるPDCAサイクルの考え方が有効です。
*   **④ （誤り）**: BCM/BCPの策定においては、**原因事象**（地震、水害、感染症など）ではなく、それによって引き起こされる**結果事象**（拠点使用不能、従業員の出社困難、サプライチェーンの途絶、システムダウンなど）に着目して対応策を検討することが推奨されています。なぜなら、原因は無数に考えられますが、ビジネスに与える影響（結果）は限られたパターンに集約できるからです。結果事象で考えることで、未知の脅威にも対応できる、より網羅的で実用的な計画を策定できます。記述はこの考え方と**逆**のことを述べているため誤りです。

> この問題は、システムの信頼性に関する計算問題です。(i)は並列冗長システムの信頼度、(ii)はアベイラビリティから故障率を求める問題です。

*   **(i) （ケ）の解説**:
    *   1台の装置Aが正常に稼働する信頼度 R = 0.7
    *   1台の装置Aが故障する確率（不信頼度） F = 1 - R = 1 - 0.7 = 0.3
    *   n台全てが故障する確率 F_sys は、各装置が独立に故障すると仮定すると、F_sys = Fⁿ = 0.3ⁿ
    *   システム全体が正常に稼働する信頼度 R_sys は、少なくとも1台が稼働していれば良いので、全体（1）から全てが故障する確率を引いたものになります。
    *   R_sys = 1 - F_sys = 1 - 0.3ⁿ
    *   このシステム信頼度が 0.9999 以上になる条件は、
        *   1 - 0.3ⁿ ≥ 0.9999
        *   0.0001 ≥ 0.3ⁿ
        *   10⁻⁴ ≥ 0.3ⁿ
    *   両辺の常用対数（log₁₀）をとります。（不等号の向きに注意）
        *   log₁₀(10⁻⁴) ≥ log₁₀(0.3ⁿ)
        *   -4 ≥ n * log₁₀(0.3)
    *   ここで、log₁₀(0.3) = log₁₀(3/10) = log₁₀(3) - log₁₀(10) = 0.477 - 1 = -0.523

        *   -4 ≥ n * (-0.523)

        *   4 / 0.523 ≤ n
        *   7.64... ≤ n

    *   MTTR: 平均修復時間 (Mean Time To Repair)

**【解説】**
> この問題は、情報セキュリティにおける利用者認証の方式とその特徴に関する正誤問題です。

*   ① （正しい）： 認証の3要素についての記述です。それぞれ「知識情報」（パスワード、PINコード）、「生体情報」（指紋、顔）、「所持情報」（ICカード、スマートフォン）と呼ばれます。これらを組み合わせることで、より強固な認証が実現できます（多要素認証）。
*   ② （正しい）： チャレンジレスポンス認証は、サーバが「チャレンジ」（ランダムなデータ）を送信し、クライアントはそれとパスワードなどから計算した「レスポンス」を返す方式です。パスワード自体はネットワーク上を流れないため、盗聴に対する耐性が高まります。
*   **③ （誤り）**: 二要素認証（2FA）や多要素認証（MFA）とは、認証の3要素（知識、所持、生体）のうち、**異なる2つ以上の要素**を組み合わせて行う認証のことです。パスワードと秘密の質問の答えは、どちらも同じ「知識情報」に分類されるため、これを2回行っても**二要素認証にはなりません**。これは多段階認証の一種ではありますが、セキュリティ強度は単一要素のままです。
*   ④ （正しい）： 生体認証のトレードオフ関係と限界について正しく述べています。他人受入率（FAR: False Acceptance Rate）と本人拒否率（FRR: False Rejection Rate）は、認証の厳しさを調整する閾値によって相反する関係にあります。セキュリティを重視して本人確認を厳しくする（FARを低くする）と、正規のユーザでも認証に失敗しやすく（FRRが高くなる）なります。この2つの率が等しくなる点をEER（Equal Error Rate）と呼び、生体認証システムの精度指標の一つとなります。両方を同時に0にすることは原理的に不可能です。
